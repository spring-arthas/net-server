# 聊天附件目录归属与历史数据恢复设计

## 目标

修复聊天图片原图、缩略图和预览图被写入 `file.parent_id=0/-1` 的问题，保证所有有效文件都挂在当前用户的真实目录节点下，且数据库记录与物理文件一一对应。历史聊天消息中的 `fileId` 保持不变，避免破坏跨 Mac 的图片查看、缓存和下载逻辑。

## 已确认现状

- 家校测试库共有 29 条有效文件记录使用非法父目录，其中 `parent_id=0` 为 11 条，`parent_id=-1` 为 18 条。
- 29 条均为聊天图片附件，分别被 `user_friend_message.id=225..229` 引用。
- 这些记录的 `user_id` 均为空，但 `user_name` 可映射到用户 7（18806504525）和用户 15（15757821544）。
- 物理文件当前位于 `/Users/hljy/Documents/storages/2026/07/02`，没有进入用户根目录。
- 28 个文件的数据库大小与物理大小一致；文件 `7471510911010766848` 是损坏的预览图，物理内容包含重复 JPEG 起始段且没有完整结束标记，需要从对应原图重新生成。
- 根因在 chat-storage：聊天原图默认上传到目录 0，派生缩略图和预览图上传到目录 -1；net-server 旧逻辑直接把客户端 `dirId` 写入 `file_task.parent_id`，最终复制到 `file.parent_id`。

## 目录模型

每个用户根目录下维护一个服务端专用隐藏目录：

```text
/Users/hljy/Documents/storages/{userName}/.chat-attachments
```

数据库目录记录满足：

- `parent_id` 等于该用户根目录 ID；
- `file_name=.chat-attachments`；
- `is_file=N`、`is_exist=Y`、`del=N`；
- `user_id` 和 `user_name` 与认证用户一致；
- 该目录不返回给普通云盘目录树，且不允许客户端创建同名目录、改名、移动或删除。

聊天附件文件的 `parent_id` 必须是该隐藏目录的正数 ID。普通云盘上传仍必须提供一个属于当前用户的正数目录 ID。

## 上传协议与服务端处理

不新增或修改任何帧类型。现有上传 JSON 元数据增加可选字段：

```json
{"uploadPurpose":"CHAT_ATTACHMENT"}
```

- chat-storage 的聊天图片原图、缩略图和预览图都发送 `CHAT_ATTACHMENT`。
- 普通云盘上传默认使用 `CLOUD_FILE`，兼容未提供该字段的请求。
- net-server 先校验传输令牌，再处理上传用途。
- `CHAT_ATTACHMENT` 请求不信任客户端 `dirId`，服务端按令牌中的 `userId/userName` 查找或创建隐藏目录，并回写请求的真实 `dirId`。
- 断点检查与元数据初始化使用同一个目录解析方法，保证断点任务和新任务身份一致。
- 服务端最终创建 `file_task` 和 `file` 时只能持久化解析后的正数目录 ID。

## 历史数据迁移

迁移前生成数据库快照和文件清单。迁移按用户执行：

1. 校验用户、用户根目录、消息引用、物理文件和数据库记录。
2. 创建或复用 `.chat-attachments` 数据库目录及物理目录。
3. 将旧文件移动到对应用户隐藏目录，保留磁盘上的随机前缀文件名。
4. 更新 `file.parent_id/user_id/user_name/file_path/file_size/gmt_modified`，不修改 `file.id`。
5. 损坏的预览图从对应原图重新生成，并同步更新 `file.file_size` 与消息 JSON 中的 `previewFileSize`。
6. 校验 29 条记录的父目录均为正数、所有文件存在、大小一致、5 条消息引用的全部 fileId 可查询。
7. 仅在旧日期目录为空时删除空目录；任何未识别文件都保留并报告。

文件移动与数据库事务无法形成单一原子事务，因此迁移脚本使用补偿策略：数据库更新失败时把已移动文件移回；移动失败时不提交数据库。备份清单记录每个文件的新旧路径，可人工回滚。

## 错误处理

- 用户或根目录不存在：拒绝创建附件目录和上传。
- 隐藏目录记录属于其他用户、状态无效或路径越界：拒绝并记录安全日志。
- 隐藏目录路径被普通文件或符号链接占用：拒绝，不覆盖。
- 普通上传 `dirId<=0`：继续拒绝。
- 历史文件缺失或大小异常：不静默迁移，先恢复或列入失败清单。

## 验证

- Java 单元测试覆盖隐藏目录创建、复用、用户隔离、路径安全、保留正数 parentId，以及普通上传拒绝 0/-1。
- Swift 单元测试覆盖聊天原图和派生图统一发送 `CHAT_ATTACHMENT`，普通上传保持默认用途。
- 使用 Zulu JDK 8 运行 net-server 全量测试和 Maven 打包。
- 迁移完成后重新查询家校测试库，并逐个执行文件存在性和大小校验。
