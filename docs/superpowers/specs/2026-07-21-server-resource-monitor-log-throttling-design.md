# 服务端资源监控日志节流设计

## 目标

保留 `ServerResourceMonitor` 每 2 秒一次的资源采样与压力等级计算，避免降低上传并发控制和背压决策的响应速度；仅减少无状态变化时重复输出的 DEBUG 日志。

## 设计

- 资源采样间隔继续保持 2 秒。
- 压力等级发生变化时立即输出 WARN 日志，不受节流影响。
- 资源采样发生异常时立即输出 WARN 日志，不受节流影响。
- 压力等级未变化时，普通资源采样 DEBUG 日志最多每 30 秒输出一次。
- 第一次成功采样允许立即输出 DEBUG，便于确认监控器工作正常。
- 启动日志同时说明采样间隔和普通日志输出间隔。

## 实现边界

只修改 `ServerResourceMonitor` 的日志输出判断，不修改压力阈值、采样指标、调压策略、上传并发限制或磁盘 IO 计算方式。

## 验证

- 连续多次同等级采样时，30 秒内只输出一次 DEBUG。
- 压力等级变化时，即使距离上一条 DEBUG 不足 30 秒，也立即输出 WARN。
- 采样周期仍为 2 秒。
